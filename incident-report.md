# Security Incident Investigation Report

**Incident ID:** IR-001  
**Severity:** High  
**Category:** Identity Security Incident  
**Status:** Resolved  
**Assigned Team:** IT Security  
**Created Date:** 2026-03-10

---

## Incident Summary

A security alert was generated indicating suspicious authentication activity for a user account in Microsoft Entra ID. The alert indicated that a login attempt originated from a geographic location that differed from the user's normal sign-in pattern.

The purpose of this investigation was to determine whether the login activity represented legitimate user behavior or a potential unauthorized access attempt.

---

## Environment

Microsoft Entra ID  
Microsoft 365  
Windows 11 workstation  

---

## Detection

The alert was generated by Microsoft Entra ID Identity Protection due to a login attempt from an unfamiliar geographic location.

This triggered a risk-based authentication alert indicating potentially suspicious activity.

---

## Investigation

### Step 1 – Review Authentication Logs

Authentication logs were reviewed in Microsoft Entra ID to examine recent login activity associated with the user account.

Result  
Multiple login attempts were detected from Bucharest, Romania using an unfamiliar device.

---

### Step 2 – Compare Normal Login Patterns

Previous authentication activity for the user was reviewed to identify typical login patterns.

Result  
The user normally signs in from Seattle, Washington using a Windows 11 workstation.

---

### Step 3 – Review Device Information

Device information associated with the login attempts was analyzed.

Result  
The login attempt originated from an unknown device not previously associated with the user account.

---

### Step 4 – Verify User Activity

The user was contacted to verify whether the login attempt was legitimate.

Result  
The user confirmed they did not attempt to sign in from the location identified in the logs.

---

### Step 5 – Evaluate Risk Level

Based on authentication logs, device information, and user confirmation, the login attempt was determined to represent a potential unauthorized access attempt.

Risk Level  
High

---

## Containment Actions

The following containment actions were implemented to secure the account.

Password reset was enforced for the user account.

All active user sessions were revoked.

Multi-Factor Authentication status was verified and enforced.

Conditional Access policies were reviewed to ensure strong authentication requirements.

---

## Resolution

The user account was secured by resetting credentials and enforcing Multi-Factor Authentication.

No evidence was found that the unauthorized login attempt successfully accessed organizational resources.

---

## Lessons Learned

Monitoring authentication logs is critical for identifying suspicious login activity.

Enforcing Multi-Factor Authentication significantly reduces the risk of unauthorized access.

Prompt investigation of authentication alerts allows security teams to respond quickly to potential threats.

Implementing Conditional Access policies strengthens identity security across the organization.
