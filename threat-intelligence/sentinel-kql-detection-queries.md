# Password Spray Detection Query

## Overview

This procedure explains how a Security Operations Center analyst can run a Kusto Query Language (KQL) query inside Microsoft Sentinel to detect password spraying activity. Password spraying occurs when an attacker attempts to authenticate against multiple user accounts using a small number of common passwords.

The detection query analyzes authentication logs to identify repeated failed login attempts across many user accounts originating from the same IP address within a short timeframe.

---

# Running the Detection Query in Microsoft Sentinel

## Step 1: Sign in to the Azure Portal

1. Open a web browser.
2. Navigate to the Azure portal.

https://portal.azure.com

3. Sign in using an account with access to Microsoft Sentinel.

---

## Step 2: Open Microsoft Sentinel

1. In the Azure portal search bar at the top of the page type:

Microsoft Sentinel

2. Click:

Microsoft Sentinel

3. Select your Sentinel workspace.

---

## Step 3: Open the Logs Query Interface

1. In the Microsoft Sentinel navigation pane select:

Logs

2. The Log Analytics query editor will open.

3. Ensure the correct workspace is selected in the upper left corner.

---

## Step 4: Enter the Detection Query

1. In the query editor window paste the following query.

```kusto
SigninLogs
| where ResultType != 0
| summarize FailedAttempts = count(), TargetedUsers = dcount(UserPrincipalName) by IPAddress, bin(TimeGenerated, 15m)
| where FailedAttempts >= 15 and TargetedUsers >= 5
| order by FailedAttempts desc
```

---

## Step 5: Run the Query

1. Click the **Run** button located above the query editor.

2. Microsoft Sentinel will execute the query against authentication logs.

3. The results will appear in the results panel below the query editor.

---

## Step 6: Review the Results

Review the output columns for:

IP Address  
FailedAttempts  
TargetedUsers  
TimeGenerated  

Look for IP addresses generating a high number of authentication failures across multiple user accounts.

This behavior may indicate password spraying activity.

---

# Query Explanation

Below is a line by line explanation of the KQL query.

---

## Line 1

```
SigninLogs
```

Explanation:

This specifies the **data table** that the query will analyze.

The SigninLogs table contains authentication events generated by Microsoft Entra ID. These logs include information such as:

User account  
IP address  
Authentication result  
Application accessed  
Login timestamp  

These logs are commonly used during identity based investigations.

---

## Line 2

```
| where ResultType != 0
```

Explanation:

The pipe symbol `|` passes data to the next operation in the query.

The `where` operator filters the results.

`ResultType != 0` means the query only returns **failed authentication attempts**.

In Microsoft Entra ID logs:

ResultType = 0 → Successful login  
ResultType ≠ 0 → Failed login

This step isolates failed authentication attempts, which are important when detecting credential attacks.

---

## Line 3

```
| summarize FailedAttempts = count(), TargetedUsers = dcount(UserPrincipalName) by IPAddress, bin(TimeGenerated, 15m)
```

Explanation:

The `summarize` operator aggregates log data to identify patterns.

This line performs several actions:

FailedAttempts = count()

Counts the total number of failed login attempts.

TargetedUsers = dcount(UserPrincipalName)

Counts the number of **unique user accounts targeted** by the attack.

`dcount()` stands for **distinct count**.

by IPAddress

Groups authentication attempts by the source IP address.

bin(TimeGenerated, 15m)

Groups the activity into **15 minute time windows**.

This allows analysts to identify bursts of authentication activity within short timeframes.

---

## Line 4

```
| where FailedAttempts >= 15 and TargetedUsers >= 5
```

Explanation:

This filter removes normal authentication activity and highlights suspicious patterns.

The query only returns results where:

At least 15 failed login attempts occurred  
AND  
At least 5 different user accounts were targeted

This pattern is typical of password spraying attacks.

Attackers attempt the same password across many accounts.

---

## Line 5

```
| order by FailedAttempts desc
```

Explanation:

This sorts the query results.

`desc` means **descending order**.

The IP addresses with the highest number of failed authentication attempts appear at the top of the results.

This allows analysts to quickly identify the most suspicious activity.

---

# How Analysts Use the Results

Once suspicious IP addresses are identified, analysts typically perform the following actions:

1. Investigate the IP address reputation.
2. Review the targeted user accounts.
3. Check whether any authentication attempts succeeded.
4. Determine whether the login activity is malicious.
5. Initiate incident response procedures if compromise is suspected.

# Conclusion

Password spraying attacks attempt to exploit weak passwords across multiple accounts. By analyzing authentication logs using KQL queries in Microsoft Sentinel, security analysts can identify abnormal login patterns and quickly investigate potential credential attacks.
