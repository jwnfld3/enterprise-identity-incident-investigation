# Data Exfiltration Detection

Detection Type: Data Security Detection  
Platform: Microsoft Sentinel  
Query Language: Kusto Query Language (KQL)

## Overview

Data exfiltration occurs when an attacker transfers sensitive data outside the organization's environment without authorization.

Attackers may attempt to download or move large amounts of data from cloud storage services such as SharePoint or OneDrive.

## Purpose

The purpose of this detection is to identify abnormal file access or download activity that may indicate unauthorized data transfer.

Early detection helps security teams prevent sensitive information from leaving the environment.

## Data Sources

The following log sources are commonly used:

- Microsoft 365 Activity Logs
- OfficeActivity table in Microsoft Sentinel
- SharePoint file access logs
- OneDrive activity logs

## Detection Logic

Indicators of potential data exfiltration include:

- large numbers of file downloads
- repeated file access within a short timeframe
- unusual user activity patterns
- downloads occurring from unfamiliar IP addresses

## Click by Click Learning Process

1. Open the Microsoft 365 Defender portal.
2. Navigate to Audit Logs.
3. Search for the user account associated with the suspicious activity.
4. Filter activity results for file related operations such as FileAccessed or FileDownloaded.
5. Identify large file downloads or access to multiple sensitive files.
6. Review timestamps and IP addresses associated with the activity.
7. Determine whether the activity deviates from the user’s normal behavior.
8. Document the activity and flag the event for investigation.

## Detection Query

```kql
OfficeActivity
| where Operation == "FileDownloaded"
| summarize DownloadCount = count() by UserId, bin(TimeGenerated, 30m)
| where DownloadCount > 20
| order by DownloadCount desc
```

This query identifies users downloading an unusually large number of files within a short timeframe.

## Investigation Process

1. Review the alert generated by the detection rule within Microsoft Sentinel.
2. Identify the user account associated with the alert.
3. Review authentication activity for the user in Microsoft Entra ID Sign in Logs.
4. Verify the source IP address and geographic location of the login activity.
5. Determine whether the authentication event originated from a known device or location.
6. Review Microsoft 365 audit logs to identify file access or download activity.
7. Identify the files that were accessed or downloaded during the timeframe of the alert.
8. Determine whether the accessed files contain sensitive organizational information.
9. Confirm whether the activity aligns with the user's normal job responsibilities.
10. Document the investigation findings and determine whether the activity represents potential data exfiltration.
