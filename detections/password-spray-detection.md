# Password Spray Detection

Detection Type: Identity Threat Detection  
Platform: Microsoft Sentinel  
Query Language: Kusto Query Language (KQL)

## Overview

Password spray attacks occur when an attacker attempts to authenticate to many accounts using a small set of commonly used passwords. This approach helps attackers avoid account lockout protections while increasing the likelihood of discovering weak credentials.

Attackers typically target enterprise environments where large numbers of user accounts exist.

## Purpose

The purpose of this detection is to identify authentication patterns consistent with password spray attacks.

Early detection allows security teams to prevent attackers from gaining access to user accounts.

## Data Sources

The following log sources are commonly used:

- Microsoft Entra ID Sign-in Logs
- Azure AD SigninLogs table in Microsoft Sentinel
- Authentication activity logs

## Detection Logic

Indicators of potential password spray activity include:

- multiple failed logins across many user accounts
- repeated login attempts from the same IP address
- authentication attempts using common passwords

## Click by Click Learning Process

1. Open the Microsoft Entra Admin Center.
2. Navigate to Monitoring.
3. Select Sign-in Logs.
4. Filter results for Failed sign-in attempts.
5. Identify repeated authentication attempts from a single IP address.
6. Review the list of targeted user accounts.
7. Determine whether the activity matches password spray patterns.
8. Document the activity and escalate the event for investigation.

## Detection Query

```kql
SigninLogs
| where ResultType != 0
| summarize FailedAttempts = count() by IPAddress, bin(TimeGenerated, 30m)
| where FailedAttempts > 20
| order by FailedAttempts desc
```

This query identifies IP addresses responsible for a high number of failed login attempts.

## Investigation Process

1. Review the alert generated by the detection rule.
2. Identify the IP address associated with the login attempts.
3. Review authentication logs for affected user accounts.
4. Determine the number of accounts targeted.
5. Review login timestamps to confirm spray behavior.
6. Check whether any login attempts were successful.
7. Identify geographic location of the IP address.
8. Determine whether the activity originates from known infrastructure.
9. Document investigation findings.
10. Escalate the incident if compromise is suspected.
