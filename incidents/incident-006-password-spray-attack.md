# Incident Report: Password Spray Attack

## Case Metadata

Incident ID: IR-006  
Category: Credential Attack  
Severity: High  
Status: Resolved  

Detection Source: Microsoft Entra ID Sign-in Monitoring  
Reported By: Automated Security Alert  

Investigation Start Time: 2026-03-20 02:05 UTC  
Investigation End Time: 2026-03-20 03:00 UTC  

Affected Systems: Microsoft 365 Authentication Services  
Targeted Accounts: Multiple user accounts  

---

## Incident Summary

Security monitoring detected a series of authentication attempts targeting multiple user accounts within the Microsoft 365 environment.

The login attempts originated from a single external IP address and attempted authentication against several user accounts using the same password.

This pattern is consistent with a **password spray attack**, where attackers attempt to authenticate using common passwords across many accounts.

---

## Detection

The incident was detected through authentication monitoring alerts generated by Microsoft Entra ID.

Indicators included:

- Multiple failed authentication attempts across several accounts
- Repeated login attempts from a single IP address
- Account lockout events triggered across multiple users

---

## Investigation

### Step 1 – Review Authentication Logs

Security investigators reviewed authentication logs for the affected accounts.

Evidence source: Microsoft Entra ID Sign-in Logs

Investigators analyzed authentication telemetry including login timestamps, IP address data, and authentication results.

Log analysis confirmed repeated login attempts targeting multiple user accounts originating from a single external IP address.

Each login attempt used the same password value across different accounts, which is a typical indicator of a password spray attack.

---

### Step 2 – Identify Attack Pattern

Investigators examined the sequence of authentication attempts to identify patterns associated with credential attacks.

Observed indicators included:

- Authentication attempts targeting multiple accounts
- A single IP address responsible for repeated login attempts
- Identical password attempts across multiple user accounts
- Multiple authentication failures occurring within a short time period

These indicators confirmed that the activity matched the behavior commonly associated with password spraying attacks.

---

### Step 3 – Review Account Lockout Events

Security investigators reviewed account lockout logs to determine the impact of the attack.

Several targeted user accounts triggered account lockout protections after repeated failed login attempts.

Account lockout policies successfully prevented further authentication attempts after the threshold for failed logins was reached.

---

### Step 4 – Evaluate Security Controls

Investigators verified that existing identity security controls functioned as intended.

Security controls confirmed:

- Account lockout policies successfully blocked repeated login attempts
- Multi Factor Authentication remained enabled for affected accounts
- Conditional Access policies were actively enforcing authentication requirements

These controls prevented attackers from successfully authenticating to the environment.

---

## Remediation

The following remediation actions were taken to mitigate the attack and secure affected accounts.

- Suspicious IP address activity was documented and monitored
- Targeted user passwords were reset
- Account lockout policies were reviewed
- Multi Factor Authentication enforcement verified
- Security monitoring increased for additional authentication anomalies

These remediation steps ensured that the attempted credential attack did not result in unauthorized access.

## Lessons Learned

This investigation highlights how password spray attacks can target multiple user accounts simultaneously using commonly used passwords.

Strong identity protection measures such as Multi Factor Authentication and account lockout policies significantly reduce the likelihood of successful credential attacks.

Continuous monitoring of authentication logs allows security teams to detect suspicious login patterns and respond quickly to credential-based attacks.

Organizations should maintain strong password policies and enforce additional authentication protections to prevent credential abuse.
