## Incident Summary

An "Impossible Travel" alert was generated by Microsoft Entra ID Identity Protection indicating that a user authenticated from two geographically distant locations within an unrealistic time frame.

Authentication logs showed the user successfully signing in from Seattle, United States, followed shortly by another login attempt from Frankfurt, Germany.

The travel time between these locations made legitimate user activity unlikely.

---

## Detection

The incident was detected automatically through Microsoft Entra ID Identity Protection risk monitoring.

Indicators included:

- Successful login from Seattle, USA
- Successful login from Frankfurt, Germany within minutes
- Unusual login pattern inconsistent with normal user behavior
